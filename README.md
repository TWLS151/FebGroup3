## 📌 스터디 목적
- 알고리즘 문제 풀이 능력 향상 및 코드 리뷰를 통한 협업 역량 강화
- 주요 알고리즘/자료구조 개념을 실전 문제로 체득

## 🗓 진행 방식
1. 매일 지정된 문제를 풀기
2. 각자 1~2문제씩 풀이 및 PR 제출 -> 이때 서로 합의한 문제 대해서는 코드 발표를 준비해야함
3. 정해진 시간에 코드 리뷰 및 풀이 토론 (30–60분)

## 🧭 협업 규칙
- 브랜치 규칙: `feature/{이름}-{문제번호}` (예: `feature/hyw-2108`)
- PR 제목: `[문제번호] 문제제목 (이름)`
- 코드 리뷰: 최소 1명 이상 승인 필요

## 📝 문제 풀이 규칙
- 문제 코드는 `problems/{번호}-{이름}` 폴더에 저장
  - 예: `problems/2108-수열/solution.py`
- 제출 전 로컬에서 기본 테스트 및 경계 케이스 확인
- 시간 복잡도와 사용한 알고리즘/아이디어를 `README.md`나 코드 상단 주석으로 명시

## 🔧 코드 스타일 & 커밋 메시지
- 언어별 기본 스타일을 준수 (Python: PEP8)
- 커밋 메시지 형식: `feat: 문제번호 - 간단한 설명 (이름)`

## 📚 자료 및 레퍼런스
- 알고리즘 문제 사이트: Baekjoon, LeetCode, Programmers
- 추천 교재/블로그 링크는 `resources/` 폴더에 정리

## 🤝 기여 방법
1. 이 레포지토리를 Clone
2. 브랜치 생성 후 작업 → PR 생성
3. 코드/설명 보완 요청 시 반영
